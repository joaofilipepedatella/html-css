usando links externos

Você pode acessar um vídeo repositor de energias saudáveis

Mas para ter um efeito supremo, é melhor acessar locais melhores, mesmo que pagos

âncora = hyperlink

A Tag A <a> é utilizada para criar a âncora.
Nela possui o Parâmetro href onde você coloca a URL no local do valor, site esse que será encaminhado após o click.

Por exemplo:
<a href="http://www.youtube.com/">

Caso queira que a pessoa que clicar no link ancorado abra uma nova janela, dentro da tag A, após o href, você deve adicionar os pares de Parâmetro-Valor:
target=_"blank"
rel="external"
Assim o cliente que clicar no hyperlink, abre uma nova página, sem sair da principal que ele já está.

Por exemplo:
<a href="http://www.youtube.com/" target="_blank" rel="external">

Usando links internos

Para acessar uma página diferente dentro do próprio site, você precisa criar um segundo arquivo no VSCode.
A tag continua sendo o <a>, porém o parâmetro href será o novo arquivo que você criou para a página a ser acessada. Por exemplo:
<a href="pagina002.html" target="self" rel="next">

Clique Aqui para acessar a Página 002

Clique Aqui para acessar a Página 003 que está dentro da pasta Notícias

o parâmetro "target" não é obrigatório, pois está no mesmo site, a menos que queira abrir uma aba separada, usando o _blank como valor.

Caso a página a ser acessada esteja em uma outra pasta dentro do projeto, uma subpasta, o caminho em href deve ser completado para o acesso ser concluído, por exemplo:
<a href="noticias/pagina002.html" target="self" rel="next">

Links de Download

Para criar um link de download, você deve criar uma pasta dentro do projeto para armazenar os arquivos a serem baixados

Criar uma lista para isso pode ser uma boa estética, caso haja mais de um link na página.

Usar a TAG A normalmente, e no parametro href colocar como valor o caminho do arquivo, mostrando a pasta.
em alguns casos, o navegador pode abrir um PDF em uma nova aba ao inves de baixá-lo. No site iana.org você encontra uma lista de códigos que podem auxiliá-lo com um novo parametro, o TYPE.
